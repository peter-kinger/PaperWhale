# 自定义Prompt编写指南

## 概述

本工具支持完全自定义的Prompt配置，你可以：
1. **选择预设模板** - 使用内置的多种分析框架
2. **编辑现有模板** - 修改预设模板的内容
3. **创建全新模板** - 完全按需编写专属Prompt

---

## 占位符说明

编写Prompt时，可使用以下占位符（变量）：

| 占位符 | 说明 | 示例 |
|:---|:---|:---|
| `{filename}` | PDF文件名 | "paper.pdf" |
| `{title}` | 论文标题 | "Climate Change and Society..." |
| `{authors}` | 作者信息 | "Zhang, Wang, Li" |
| `{year}` | 论文年份 | "2025" |
| `{paper_content}` | 论文全文内容 | (论文的完整文本) |

## 内置模板

### 1. default - 默认-深度分析
完整的学术论文分析，包含：
- 工作讲解
- 研究总结（动机、方法、数据、局限、拓展、大纲）
- 图/表讲解
- 核心贡献与拓展思考

### 2. quick_scan - 快速扫描
适合大批量初筛，输出简洁：
- 研究问题
- 主要方法
- 核心结论
- 局限性

### 3. methodology - 方法论聚焦
重点分析研究方法：
- 方法概述
- 技术路线（理论框架、模型构建、数据处理、实验设计）
- 方法创新
- 方法评估与改进建议

### 4. critical_review - 批判性评审
严厉但公正的评审：
- 研究问题评估
- 方法学评估
- 结果解释评估
- 论文写作评估
- 总体评分与改进建议

### 5. abm_modeling - ABM建模专项
针对多主体建模研究：
- 建模概述
- 主体设计（类型、属性、交互）
- 规则与机制
- 模型验证与分析
- 代码与数据
- 局限与拓展

### 6. climate_modeling - 气候模型专项
针对气候科学研究：
- 研究背景
- 模型与方法
- 数据与验证
- 主要发现
- 不确定性
- 政策意义

## 如何创建自定义Prompt

### 示例1：适合写文献综述的Prompt

```
你是一位学术研究员，请为撰写文献综述提供帮助。仔细阅读以下论文并提供分析：

标题: {title}
作者: {authors}
年份: {year}

内容:
{paper_content}

请按以下格式输出：

**1. 研究主题归类**
这篇论文属于哪个具体的研究领域？关键词有哪些？

**2. 核心观点提炼**
作者的主要论点是什么？用3-5句话概括。

**3. 理论框架**
论文基于什么理论？使用了什么概念？

**4. 方法论**
采用了什么研究方法？数据来源是什么？

**5. 与其他文献的关系**
这篇论文与该领域其他研究有什么联系？支持还是挑战了现有观点？

**6. 可引用观点**
论文中最有价值且适合引用的3个观点是什么？请给出原文引用。
```

### 示例2：适合技术实现的Prompt

```
你是AI领域的算法工程师，请分析以下论文的技术实现细节：

标题: {title}
作者: {authors}
年份: {year}

内容:
{paper_content}

请详细分析：

**1. 问题定义**
论文要解决什么具体问题？问题的输入输出是什么？

**2. 算法流程**
请逐步描述算法的实现流程，用自然语言解释每个步骤。

**3. 核心公式**
论文中的核心数学公式是什么？请解释每个变量的含义。

**4. 代码架构**
如果让你实现这个算法，你会如何组织代码结构？列出主要模块。

**5. 复杂度分析**
算法的时间复杂度和空间复杂度是多少？

**6. 可复现性**
论文是否提供了足够的细节来复现结果？缺少什么信息？
```

### 示例3：适合对比分析的Prompt

```
请对比分析以下论文，并提供结构化的比较报告：

标题: {title}
作者: {authors}
年份: {year}

内容:
{paper_content}

请提供以下维度的对比分析：

**1. 研究问题对比**
本文研究与该领域主流研究在问题上有什么异同？

**2. 方法对比**
本文方法与同类方法相比，有什么优势和劣势？

**3. 数据对比**
使用的数据集、样本量、数据质量等方面如何？

**4. 结果对比**
得出的结论是否新颖？与同类研究结论是否一致？

**5. 贡献度评估**
本文在以下评分维度的得分（1-10分）：
- 创新性：__分
- 严谨性：__分
- 实用性：__分
- 可复现性：__分

请说明每项评分的依据。
```

## 交互式创建自定义Prompt

运行程序后选择 `[3] Prompt模板管理` -> `[n] 创建新的自定义模板`

```
请按提示操作：

模板名称: 我的文献综述Prompt
模板描述: 适合撰写系统性文献综述的分析框架

--- 输入 paper_analysis Prompt ---
[输入你的完整Prompt内容]
[输入完成后按回车换行，再输入 END 结束]

--- 输入 brief_summary Prompt ---
[简短摘要Prompt]
```

## 命令行自定义Prompt

```bash
# 使用临时自定义Prompt
python main.py --input ./input_pdfs --custom-prompt "你的Prompt内容"

# 使用指定模板
python main.py --input ./input_pdfs --prompt abm_modeling

# 列出所有模板
python main.py --list-prompts
```

## 最佳实践

1. **保持结构清晰**：Prompt结构要清晰，但输出结果要避免过度使用标题符号
2. **控制长度**：避免Prompt过长导致内容被截断
3. **领域适配**：根据研究领域选择或创建合适的模板
4. **迭代优化**：根据实际输出效果不断调整Prompt
5. **保存备份**：重要的自定义Prompt建议备份到文件
